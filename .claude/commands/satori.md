description: Внести ФАКТ из месячного отчёта «Сатори» (агентское вознаграждение Uzum) в эталон и дашборд, задеплоить
---

Внеси фактические деньги из месячного отчёта «Сатори» и задеплой. Только данные, дизайн не трогать.

1. **Найди отчёт.** Месячный акт «Отчёт о выполнении агентского поручения» (UZUM ↔ SATORI IT) — **PDF** (текущий формат Uzum) либо `.docx` (старый). Часто приходит в Telegram → `Downloads\Telegram Desktop\` (напр. `Отчет_ООО_SATORI_IT <месяц>.pdf`); может лежать и в корне проекта как `сатори_ГГГГ-ММ.pdf`/`.docx`. Конфиденциальный, в `.gitignore`. Если файла нет — остановись и сообщи.

2. **Запусти:**
   ```
   python build_satori.py "<путь к отчёту .pdf|.docx>" Эталон_ПВЗ.xlsx index.html
   ```
   Скрипт: авто-определяет формат по расширению (`.pdf` → pdfplumber, `.docx` → таблицы Word), парсит доход ПВЗ = фикс. вознаграждение [гарантия, прорейчено] + п.1.8 + компенсация п.5.2.1, определяет месяц, **накопительно** пишет лист `Сатори_факт` в эталон (закрытые месяцы не переписывает, текущий обновляет), сверяет ИТОГО, пишет `FACT` (все накопленные месяцы) в `data.json` (ключи `DATA`/`WK`/`SAL` не трогает). `index.html` третьим аргументом — только для определения папки `data.json`.
   - Если эталон открыт в Excel → ошибка доступа: попроси закрыть файл.
   - Если коды ПВЗ нет в Справочнике → скрипт сообщит и пропустит; добавь строку в «Справочник» и перезапусти.
   - Покажи вывод (месяцев накоплено, ИТОГО дохода за месяц, число ПВЗ).

3. **Синхронизируй копию** index.html → Uzum_дашборд.html (проверь хэш).

4. **Проверки:** (а) `data.json` — валидный JSON, ключ `FACT` содержит нужный месяц, `DATA`/`WK`/`SAL` на месте: `"C:\Program Files\nodejs\node.exe" -e "const d=JSON.parse(require('fs').readFileSync('data.json'));console.log(Object.keys(d.FACT.months), Object.keys(d))"`. (б) `node --check` на инлайн-`<script>`. **Обязательно визуальная проверка**: данные теперь грузятся через `/data.js` за паролем, поэтому подними локально настоящий `server.js` (`PORT=5080 DASH_PASS=… node server.js`) и зайди по паролю, ЛИБО статик-сервер, который вместе с `index.html` отдаёт `/data.js` (читает `data.json` и оборачивает в `const DATA=…,WK=…,FACT=…,SAL=…`). На «Обзоре» KPI «P&L по сети / P&L/точка» должны показывать факт с бейджем «факт · Сатори · <месяц>», а при движении симулятора — переключаться на модель. Снимок + сверка значений.

5. **Деплой:** `git add -A && git commit -m "satori: факт <месяц>" && git push origin main`. Отчёт (`.pdf`/`.docx`) в коммит не попадёт (gitignore). Эталон коммитится.

Логика факта (не менять без просьбы): доход ПВЗ — из Сатори; затраты — из Справочника, прорейчены по активным дням; факт-P&L = доход − затраты; факт замещает модель в P&L по умолчанию, модель остаётся для симулятора/прогноза.
