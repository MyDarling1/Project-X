description: Внести ФАКТ из месячного отчёта «Сатори» (агентское вознаграждение Uzum) в эталон и дашборд, задеплоить
---

Внеси фактические деньги из месячного отчёта «Сатори» и задеплой. Только данные, дизайн не трогать.

1. **Найди отчёт.** Ожидается `сатори_ГГГГ-ММ.docx` в корне проекта (Word-акт «Отчёт о выполнении агентского поручения», UZUM ↔ SATORI IT). Конфиденциальный, в `.gitignore`. Если файла нет — остановись и сообщи.

2. **Запусти:**
   ```
   python build_satori.py сатори_ГГГГ-ММ.docx Эталон_ПВЗ.xlsx index.html
   ```
   Скрипт: парсит таблицу (доход ПВЗ = фикс. вознаграждение [гарантия, прорейчено] + п.1.8 + компенсация п.5.2.1), определяет месяц, **накопительно** пишет лист `Сатори_факт` в эталон (закрытые месяцы не переписывает, текущий обновляет), сверяет ИТОГО, вшивает `FACT` (последний месяц) в `index.html`.
   - Если эталон открыт в Excel → ошибка доступа: попроси закрыть файл.
   - Если коды ПВЗ нет в Справочнике → скрипт сообщит и пропустит; добавь строку в «Справочник» и перезапусти.
   - Покажи вывод (месяцев накоплено, ИТОГО дохода за месяц, число ПВЗ).

3. **Синхронизируй копию** index.html → Uzum_дашборд.html (проверь хэш).

4. **Проверки:** `node --check` на инлайн-`<script>`; убедись, что есть `const FACT = {`, `const DATA = [`, `const WK = {`. **Обязательно визуальная проверка**: подними превью, на «Обзоре» KPI «P&L по сети / P&L/точка» должны показывать факт с бейджем «факт · Сатори · <месяц>», а при движении симулятора — переключаться на модель. Снимок + сверка значений.

5. **Деплой:** `git add -A && git commit -m "satori: факт <месяц>" && git push origin main`. Отчёт `.docx` в коммит не попадёт (gitignore). Эталон коммитится.

Логика факта (не менять без просьбы): доход ПВЗ — из Сатори; затраты — из Справочника, прорейчены по активным дням; факт-P&L = доход − затраты; факт замещает модель в P&L по умолчанию, модель остаётся для симулятора/прогноза.
