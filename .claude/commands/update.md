description: Обновить данные дашборда из узум.xlsx (накопление недель) и задеплоить в main
---

Обнови данные дашборда Uzum и задеплой. Выполняй строго по шагам и не меняй дизайн/логику — только данные.

1. **Найди дневной файл.** Ожидается `узум.xlsx` в корне проекта (это OneDrive online-only плейсхолдер — может не показываться в фильтрованных листингах; ищи через сырой перебор каталога / `.NET enumeration`, не путай с `Эталон_ПВЗ.xlsx`). Если файла нет — остановись и сообщи об этом, ничего не меняй.

2. **Запусти накопительное обновление:**
   ```
   python build_data.py узум.xlsx Эталон_ПВЗ.xlsx index.html
   ```
   Скрипт накапливает недели в `Эталон_ПВЗ.xlsx` (полные не трогает, последнюю неполную обновляет, новые дописывает) и вшивает историю в `index.html`.
   - **Если скрипт упал из-за формата файла** (не те колонки `Адрес/ПВЗ/Дата/Кол-во заказов/Общая сумма` или нет листов-дат ДД.ММ.ГГГГ) — **остановись**, покажи точную ошибку, данные руками НЕ подгоняй.
   - Если в Справочнике нет нового ПВЗ из файла — скрипт его пропустит и сообщит; тогда сообщи пользователю (строку ПВЗ в «Справочник» добавляет он).
   - Покажи вывод скрипта (сколько недель накоплено, run-rate-неделя, дни последней недели).

3. **Синхронизируй копию:** скопируй `index.html` в `Uzum_дашборд.html` (должны быть идентичны; проверь хэш).

4. **Проверка синтаксиса:** извлеки содержимое инлайн-тега `<script>` из `index.html` и прогони `node --check` (Node: `C:\Program Files\nodejs\node.exe`). Убедись, что в `index.html` есть `const DATA = [` и `const WK = {`, число точек ПВЗ совпадает со Справочником, JSON валиден. Если проверка не прошла — остановись и покажи ошибку.

5. **Деплой (без вопросов, без PR):**
   ```
   git add -A
   git commit -m "data update: <диапазон новых/обновлённых недель>"
   git push origin main
   ```
   `узум.xlsx` не попадёт в коммит (в `.gitignore`). После пуша Railway пересоберёт сайт автоматически.

6. **Отчитайся кратко:** новые недели, обновлённая последняя неделя (дни), run-rate-неделя, что задеплоено.
