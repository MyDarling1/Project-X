# Промт для Claude Code — обновить дашборд из нового файла

Когда просто бросаешь новый дневной файл `узум.xlsx` в папку проекта (а не грузишь через портал),
открой Claude Code в папке `Project X` и вставь текст ниже (скопируй блок целиком).

---

## ВСТАВИТЬ В CLAUDE CODE:

```
Я положил в эту папку новый/обновлённый дневной файл выгрузки Uzum (xlsx с листами-датами
ДД.ММ.ГГГГ и колонками: Адрес, ПВЗ, Дата, Кол-во заказов, Общая сумма).

Сначала прочитай HANDOVER.md целиком и строго соблюдай раздел «Дизайн-система»:
ничего в оформлении дашборда не меняй (цвета, шрифты, компоненты, тёмную тему).
Твоя задача — только обновить ДАННЫЕ, не трогая дизайн и логику.

Сделай по шагам:

1. Найди дневной файл (например `узум.xlsx`; если имя другое — самый свежий .xlsx
   с листами-датами в корне папки). Не путай его с `Эталон_ПВЗ.xlsx`.

2. Запусти обновление с НАКОПЛЕНИЕМ:
       python build_data.py <дневной_файл.xlsx> Эталон_ПВЗ.xlsx index.html
   Скрипт сам:
   - накопит недели в Эталон_ПВЗ.xlsx (полные недели не перезаписывает; последнюю неполную —
     обновляет; новые недели дописывает);
   - вшьёт в index.html всю историю недель (страница «Недельная история») и пересчитает «Обзор».
   Покажи мне вывод скрипта (сколько недель накоплено, какая неделя run-rate, дни последней недели).

3. Пересчитай формулы эталона (если установлен LibreOffice):
       python /путь/к/recalc.py Эталон_ПВЗ.xlsx        # либо просто открой файл в Excel один раз
   Если recalc недоступен — пропусти, формулы пересчитаются при первом открытии в Excel.

4. Проверь, что дашборд не сломан:
   - вытащи содержимое тега <script> из index.html и прогони `node --check` (синтаксис);
   - убедись, что в index.html есть `const DATA = [` и `const WK = {`, и что число точек = 18
     (или сколько сейчас ПВЗ в Справочнике).

5. Синхронизируй копию:  скопируй index.html в Uzum_дашборд.html (они должны быть идентичны).

6. Покажи краткое резюме изменений (новые недели, обновлённая последняя неделя) и СПРОСИ меня,
   коммитить ли. Если я скажу да:
       git add -A
       git commit -m "data update: <диапазон новых недель>"
       git push
   После пуша Railway пересоберёт сайт автоматически.

Важно:
- НЕ меняй CSS/шрифты/тему/компоненты дашборда. Любые правки UI — только через существующие
  классы и CSS-переменные (см. HANDOVER §5).
- НЕ перезаписывай вручную полные недели в эталоне — этим занимается скрипт по правилу накопления.
- Если формат файла не совпал — скрипт остановится с понятной ошибкой; покажи её мне и НЕ
  подгоняй данные руками, просто сообщи, что именно не так с колонками/листами.
- Если в Справочнике нет какого-то нового ПВЗ из файла — скрипт его пропустит и сообщит;
  тогда добавь строку этого ПВЗ в лист «Справочник» (код, AKA, город, сегмент, комиссия, опекс,
  УК, гарантия, прогноз, Яндекс-пик, старт) и перезапусти шаг 2.
```

## Короткий вариант (если лень читать длинный)

```
Новый узум.xlsx в папке. Прочитай HANDOVER.md и не меняй дизайн.
Запусти: python build_data.py узум.xlsx Эталон_ПВЗ.xlsx index.html
Затем скопируй index.html в Uzum_дашборд.html, прогони node --check на <script>,
покажи вывод и спроси, коммитить ли. Если да — git add/commit/push (Railway пересоберёт).
```
