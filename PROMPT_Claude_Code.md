# Промт для Claude Code — обновить дашборд из нового файла (актуальная версия)

Когда просто бросаешь новый дневной файл `узум.xlsx` в папку проекта (а не грузишь через портал),
открой Claude Code в папке `Project X` и вставь текст ниже (скопируй блок целиком).

**ВАЖНО:** Перед любой работой прочитай полностью `HANDOVER.md` и `CLAUDE.md`.

---

## ВСТАВИТЬ В CLAUDE CODE:

```
Я положил в эту папку новый/обновлённый дневной файл выгрузки Uzum (xlsx с листами-датами
ДД.ММ.ГГГГ и колонками: Адрес, ПВЗ, Дата, Кол-во заказов, Общая сумма).

Сначала прочитай HANDOVER.md целиком и CLAUDE.md целиком. Строго соблюдай раздел «Дизайн-система»:
ничего в оформлении дашборда не меняй (цвета, шрифты, компоненты, тёмную тему).
Твоя задача — только обновить ДАННЫЕ через накопление, не трогая дизайн и логику.

Текущая архитектура:
- Данные живут в `data.json` (`DATA`, `WK`, `FACT`, `SAL`).
- `index.html` НЕ содержит данные — грузит их через `<script src="/data.js">`.
- `server.js` отдаёт `/data.js` (и сам `index.html`) только после входа по паролю из env DASH_PASS.
- Все обновления идут через Python-скрипты накопления (build_data.py / gsheet_sync.py / build_satori.py). Они пишут в `data.json` и Эталон.

Сделай по шагам:

1. Найди дневной файл (например `узум.xlsx`; если имя другое — самый свежий .xlsx
   с листами-датами в корне папки). Не путай его с `Эталон_ПВЗ.xlsx`.
   Альтернатива (основной путь): используй gsheet_sync (см. .claude/commands/sync.md).

2. Запусти обновление с НАКОПЛЕНИЕМ:
       python build_data.py <дневной_файл.xlsx> Эталон_ПВЗ.xlsx index.html
   или для Google-таблицы:
       python gsheet_sync.py __gsheet.xlsx Эталон_ПВЗ.xlsx index.html
   Скрипт сам:
   - накопит недели в Эталон_ПВЗ.xlsx (полные недели не перезаписывает; последнюю неполную —
     обновляет; новые недели дописывает);
   - пишет результат в `data.json` (DATA/WK + накопление SAL). В index.html ничего не вшивается.
   Покажи мне вывод скрипта (сколько недель накоплено, какая неделя run-rate, дни последней недели, обновлённая последняя неполная).

3. (Для Сатори) При необходимости внеси факт:
       python build_satori.py сатори_ГГГГ-ММ.docx Эталон_ПВЗ.xlsx index.html
   Факт идёт в FACT в data.json (модель остаётся для симулятора).

4. Проверь, что ничего не сломано (обязательно перед коммитом):
   - Выполни node --check на инлайн-`<script>` из index.html (синтаксис).
     На Windows используй ПОЛНЫЙ путь:
     "C:\Program Files\nodejs\node.exe" --check index.html   (или вытащи содержимое script и проверь).
   - Проверь data.json (полный путь node):
     "C:\Program Files\nodejs\node.exe" -e "const d=JSON.parse(require('fs').readFileSync('data.json'));console.log(d.DATA.length,Object.keys(d))"
   - Убедись, что ключи DATA/WK/FACT/SAL на месте, число ПВЗ совпадает со Справочником (сейчас 18).

5. Синхронизируй копию: скопируй index.html в Uzum_дашборд.html (они должны быть идентичны).

6. Сразу коммить и пуш напрямую в main (без PR и без вопроса):
       git add -A
       git commit -m "data update: <диапазон новых/обновлённых недель>"
       git push origin main
   После пуша Railway автоматически пересоберёт сайт.

Важно:
- НЕ меняй CSS/шрифты/тему/компоненты дашборда. Любые правки UI — только через существующие
  классы и CSS-переменные (см. HANDOVER §5).
- НЕ перезаписывай вручную полные недели в эталоне — этим занимается скрипт по правилу накопления.
- Если формат файла не совпал — скрипт остановится с понятной ошибкой; покажи её мне и НЕ
  подгоняй данные руками.
- Если в Справочнике нет нового ПВЗ — скрипт его пропустит; добавь строку в «Справочник» и перезапусти.
- Для визуальных правок UI (если делаешь улучшения): обязательно поднимай настоящий server.js с env (PORT=5080 DASH_PASS=... ), выполняй интерактивный аудит (прокликай ВСЕ контролы), делай скриншоты и node-проверки.
- Данные теперь за паролем — для локальной проверки рендера используй настоящий server.js или статик + обёртку data.js.
```

## Короткий вариант (если лень читать длинный)

```
Новый узум.xlsx в папке. Прочитай полностью HANDOVER.md и CLAUDE.md, не меняй дизайн.
Запусти: python build_data.py узум.xlsx Эталон_ПВЗ.xlsx index.html
(или gsheet_sync.py для Google-таблицы)

Затем:
- "C:\Program Files\nodejs\node.exe" --check на <script> из index.html
- "C:\Program Files\nodejs\node.exe" -e "..." для data.json (DATA.length и ключи)
- скопируй index.html в Uzum_дашборд.html

Сразу:
git add -A
git commit -m "data update: ..."
git push origin main
(Railway пересоберёт)
```
