# CLAUDE.md — конституция проекта (Uzum ПВЗ дашборд)

Перед любыми изменениями читай `HANDOVER.md` (полные правила: дизайн-система §5, модель данных §4, деплой §3, логика недель §8). Ниже — краткая выжимка обязательных правил.

## Git и деплой (действует всегда, во всех сессиях)

- **Всегда коммить и пушить напрямую в `main` — без pull request и без запроса подтверждения у пользователя.**
- После любых изменений сразу: `git add -A` → `git commit -m "<краткое описание>"` → `git push origin main`.
- НЕ создавай pull request.
- Push в `main` автоматически пересобирает сайт на Railway (домен `punktividachi.com`).

## Синхронизация файлов и проверки

- После правки `index.html` копируй его в `Uzum_дашборд.html` — эти два файла всегда идентичны.
- Перед коммитом прогоняй `node --check` на содержимом тега `<script>` из `index.html` (извлечь инлайн-скрипт без `src`, проверить синтаксис). Node: `C:\Program Files\nodejs\node.exe` (может не быть в PATH сессии — вызывай по полному пути).
- **Для любых ВИЗУАЛЬНЫХ изменений UI — обязательно проверяй итоговый рендер скриншотом перед коммитом** (Claude Preview: подними локальный статик-сервер и сделай скриншот нужной вкладки; пароль обходится через `localStorage.setItem('uzum_gate','uzum2026')`). `node --check` ловит только синтаксис, но не наложения подписей, переполнения и битую вёрстку.

## Дизайн — НЕ менять без явной просьбы (HANDOVER §5)

- Тёмная «операционная» тема; только заданные CSS-переменные из `:root` (`--bg`, `--panel`, `--violet`, `--mint`, `--coral`, `--s1/2/3` и т.д.).
- Шрифты: **Bricolage Grotesque** (заголовки/числа-герои), **Hanken Grotesk** (текст), **Spline Sans Mono** (цифры, класс `.mono`).
- Переиспользовать существующие компоненты: `.card`, `.kpi`, `.bebtn` (все кнопки одноцветные фиолетовые), `.seg-toggle`, `.viewnav`, `.pill`, таблицы, `.drawer`, самописные SVG-графики.
- Без новых палитр, светлых фонов, сторонних UI-библиотек, эмодзи в UI.

## Данные — НЕ менять без явной просьбы (HANDOVER §4, §7)

- Обновление данных только через накопление:
  `python build_data.py узум.xlsx Эталон_ПВЗ.xlsx index.html`
  (полные недели не перезаписываются, последняя неполная — обновляется, новые дописываются).
- `узум.xlsx` — конфиденциальная дневная выгрузка; **в репозиторий не коммитить** (в `.gitignore`). `Эталон_ПВЗ.xlsx` — источник правды, коммитится.
- НЕ редактировать вручную полные недели в эталоне — этим занимается скрипт по правилу накопления.
- НЕ переименовывать поля `DATA`/`WK` — от них зависит весь рендер. Неполные недели всегда помечать звёздочкой/штриховкой, не экстраполировать.

## Быстрая команда

- `/update` — обновить данные из `узум.xlsx` и задеплоить (см. `.claude/commands/update.md`).
