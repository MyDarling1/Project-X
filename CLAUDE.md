# CLAUDE.md — правила работы с этим репозиторием

Перед любыми изменениями читай `HANDOVER.md` (дизайн-система, модель данных, деплой).

## Деплой и git (действует всегда, во всех сессиях)

- **Всегда коммить и пушить изменения напрямую в `main` без pull request и без запроса подтверждения у пользователя.**
- После любых изменений сразу выполняй: `git add -A` → `git commit -m "<краткое описание>"` → `git push origin main`.
- НЕ создавай pull request.
- Push в `main` автоматически пересобирает сайт на Railway (домен `punktividachi.com`).

## Границы

- Дизайн (CSS/шрифты/тему/компоненты) и данные (`DATA`/`WK`, `Эталон_ПВЗ.xlsx`) НЕ меняй без явной просьбы пользователя.
- `index.html` и `Uzum_дашборд.html` держать идентичными (после правки `index.html` копировать в `Uzum_дашборд.html`).
- `узум.xlsx` — конфиденциальная дневная выгрузка, в репозиторий не коммитить (она в `.gitignore`).
